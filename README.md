## CodeKit 是什么，谁在用它

CodeKit 是一款只在 macOS 上运行的前端开发工具，由独立开发者 Bryan Jones 一个人做出来维护至今。没有团队、没有融资、就是一个人把它做成了很多大公司都在用的东西——Google 开发团队、Apple、通用电气，甚至奥巴马竞选团队当年都是它的用户。

核心逻辑很简单：把你平时要在命令行里手动做的事，全部变成一个漂亮 GUI 里的勾选框。

想压缩 CSS？打个勾。要 Babel 转译 ES6？打个勾。图片优化？一键搞定，PNG/JPEG 平均缩小 70%。

## 它能干什么

**编译一切前端语言**

Sass、Less、Stylus、CoffeeScript、TypeScript、JavaScript/ES6、Pug、Slim、Haml、Markdown、JSON、SVG……开箱即用，不需要写任何配置文件和插件。用了不在内置列表里的语言？直接在 UI 里告诉它怎么处理就行了。

**浏览器即时刷新与多设备同步**

这个功能确实有点魔法感——改完 CSS，不需要刷新浏览器，直接注入更新。更牛的是，手机、平板、Windows 电脑，只要接在同一个网络里有现代浏览器，CodeKit 全部帮你同步。连 Tesla 的浏览器官网都专门写上去了，不是 joke。

**前端优化一条龙**

图片压缩、JS 压缩合并、去除无用 CSS（PurgeCSS 集成）、缓存爆破（Cache Busting）、Source Map……这些"上线前必做清单"上的项目，CodeKit 全给你自动化了。

**内置 npm 和主流框架快速启动**

Tailwind、Bootstrap、Zurb Foundation，一键初始化，CodeKit 自己去下最新版本帮你配好。超过 130 万个 npm 包，界面操作安装，不用打命令。

**调试工具**

ESLint 内置，JSHint/JSLint 也在。找那个搞崩 IE 的悬挂逗号？用内置工具扫一遍，精确到行列数。团队协作还能强制统一代码风格，一个人用 tab 另一个人用空格的问题就此终结。

## 和命令行工具比，值得买吗

Grunt、Gulp、Webpack、Vite……这些工具当然不是"差"，只是门槛和时间成本摆在那里。Webpack 配一个项目，光 `webpack.config.js` 里各种 loader 的配置方式就能让人搜索三个小时。

CodeKit 的定位不是替代这些工具，而是**把那些"我知道该怎么做但懒得每次都重新配"的事情彻底图形化**。如果你：

- 主要做独立项目或小型网站
- 不想在每个项目里重新折腾构建工具
- Mac 用户
- 更想把时间花在写代码而不是配置上

那 CodeKit 几乎是首选。

MacUpdate 上有用户评价说，之前用 Grunt/Gulp 需要几个月才能熟悉的工作流，用 CodeKit 5 分钟就起好了项目。考虑到这是买断价格，换算成时间成本，算得过来。
